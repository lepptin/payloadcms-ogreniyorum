PayloadCMS v3 ile yeni bir projeye başlamanın resmî ve en hızlı yolu, `create-payload-app` adlı CLI aracını kullanmaktır. Bu araç, gerekli tüm bağımlılıkları ve yapılandırma dosyalarını sizin yerinize hazırlayarak sıfırdan bir Payload uygulaması ayağa kaldırır.

## Kurulum Komutu

Terminalinizde aşağıdaki komutu çalıştırın:

```bash
npx create-payload-app
```

Opsiyonel olarak en güncel sürümü garantiye almak için `@latest` etiketini ekleyebilirsiniz:

```bash
npx create-payload-app@latest
```

Bu komut, `create-payload-app` paketini geçici olarak indirip çalıştırır ve interaktif bir kurulum sihirbazı başlatır.

## Kurulum Aşamaları (Sihirbaz Adımları)

Sihirbaz sırasıyla aşağıdaki seçimleri yapmanızı ister. Her bir adım, oluşturulacak projenin temel karakteristiklerini belirler.

### 1. Proje Adı
- **Ne sorar?** `? Project name?`
- **Açıklama:** Uygulamanızın dizin adı. Bu isimle yeni bir klasör oluşturulur ve `package.json` içindeki `name` alanına yazılır. Boşluk içermeyen, npm paket ismi kurallarına uygun bir değer girin (ör. `my-payload-app`).

### 2. Veritabanı Adaptörü
- **Ne sorar?** `? Choose a database:`
- **Seçenekler (Payload v3):**
  - **SQLite (default)** – Dosya tabanlı, harici bir sunucu gerektirmez. Geliştirme ve prototipleme için idealdir. `payload.db` adlı bir dosyada verileri saklar.
  - **PostgreSQL** – Üretim ortamları için önerilen, güçlü bir ilişkisel veritabanı. Bağlantı dizesi (connection string) sonraki adımda istenir.
  - **MongoDB** – v3’te hâlâ desteklenen doküman tabanlı veritabanı. Atlas veya kendi sunucunuz üzerinden bağlanabilir.
- **Etkisi:** Seçiminiz, `payload.config.ts` içinde kullanılacak veritabanı adaptörünü ve bağlantı ayarlarını belirler. SQLite seçildiğinde ek bir bağlantı dizesi sorulmaz; PostgreSQL/MongoDB seçildiğinde bir sonraki adımda bağlantı bilgileri istenir.

### 3. TypeScript Kullanımı
- **Ne sorar?** `? Would you like to use TypeScript? (Y/n)`
- **Açıklama:** Varsayılan `Yes`’tir. Payload v3, TypeScript ile birinci sınıf destek sunar; tüm konfigürasyon, koleksiyon ve alan tanımları tip güvenliği altında yazılır. `No` seçerseniz düz JavaScript dosyaları oluşturulur, ancak tip desteğinden feragat etmiş olursunuz.

### 4. Payload Admin Paneli
- **Ne sorar?** `? Would you like to install the Payload admin panel? (Y/n)`
- **Açıklama:** Varsayılan `Yes`’tir. Admin paneli, içerik yönetimi için hazır gelen React tabanlı bir kullanıcı arayüzüdür. `Yes` dendiğinde gerekli bağımlılıklar (`payload`, `next`, `react`, `react-dom` vb.) ve panel rotaları otomatik eklenir. `No` seçerseniz yalnızca headless (API-only) bir kurulum elde edersiniz; paneli daha sonra manuel olarak ekleyebilirsiniz.

### 5. Örnek Koleksiyon Ekleme
- **Ne sorar?** `? Would you like to add an example collection? (Y/n)`
- **Açıklama:** Varsayılan `Yes`’tir. `Yes` seçildiğinde, `Posts` adında bir koleksiyon ve birkaç temel alan (başlık, içerik, yazar vb.) projeye eklenir. Bu sayede kurulumdan hemen sonra admin panelinde bir içerik tipi görür, yapıyı test edebilirsiniz. `No` dendiğinde boş bir koleksiyon yapılandırmasıyla başlarsınız.

### 6. Bağımlılıkların Yüklenmesi ve Projenin Hazırlanması
Seçimler tamamlandıktan sonra CLI:
- `package.json` oluşturur,
- Seçilen veritabanı adaptörüne göre `payload.config.ts` (veya `.js`) dosyasını hazırlar,
- Gerekli tüm npm paketlerini (`payload`, `next`, `react`, `react-dom`, veritabanı sürücüsü vb.) indirir,
- Admin paneli ve örnek koleksiyon dosyalarını oluşturur,
- `.env` dosyasına veritabanı bağlantı dizesi (SQLite dışındakiler için) ve `PAYLOAD_SECRET` gibi temel ortam değişkenlerini ekler.

Kurulum bittiğinde terminalde projeyi başlatmak için gerekli komutlar gösterilir.

## Kurulum Sonrası İlk Çalıştırma

```bash
cd <proje-adı>
npm run dev
```

Bu komut, Next.js geliştirme sunucusunu başlatır. Tarayıcınızda `http://localhost:3000` adresine giderek uygulamayı görebilir, `/admin` rotasından yönetici paneline erişebilirsiniz. İlk girişte bir admin kullanıcısı oluşturmanız istenir.

## `npx` Nedir ve Neden Avantajlıdır?

`npx`, npm (Node Package Manager) ile birlikte gelen bir **paket çalıştırıcıdır**. Bir npm paketini **global olarak kurmadan**, doğrudan çalıştırmanızı sağlar.

**`npx` olmasaydı ne yapardık?**
- Önce `create-payload-app` aracını global kurardık:
  ```bash
  npm install -g create-payload-app
  create-payload-app
  ```
- Ya da tamamen manuel bir yol izlerdik: bir Next.js projesi oluşturur (`npx create-next-app`), ardından `npm install payload` ile Payload’ı ekler, `payload.config.ts` dosyasını, admin paneli rotalarını ve veritabanı bağlantısını elle yapılandırırdık.

**`npx`’in sağladığı avantajlar:**
1. **Global kurulum gerektirmez** – Diskinizde kalıcı paket birikmesini ve olası sürüm çakışmalarını önler.
2. **Her zaman güncel sürümü kullanır** – `npx create-payload-app@latest` komutu, npm kayıt defterindeki en son sürümü geçici olarak indirip çalıştırır. Global kurulumda eski bir sürümle kalma riski vardır.
3. **Tek seferlik işlemler için idealdir** – Proje oluşturma gibi sık tekrarlanmayan görevlerde, aracı sisteminizde kalıcı olarak tutmanız gerekmez.
4. **Disk alanı ve bakım yükü azalır** – Global paketler zamanla güncellenmeyi unutabileceğiniz bir karmaşa yaratır; `npx` bu yükü ortadan kaldırır.

Özetle, `npx create-payload-app` komutu, Payload v3’ün tüm kurulum karmaşasını tek bir interaktif adımla çözerek, sizi manuel konfigürasyon detaylarından kurtarır ve dakikalar içinde çalışan bir uygulama sunar.
