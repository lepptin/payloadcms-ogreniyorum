PayloadCMS v3’te `payload.config.ts` dosyası, uygulamanızın **tüm yapısal ve davranışsal tanımlarını** bir araya getiren merkezî konfigürasyon nesnesini dışa aktarır. Mimari olarak bu dosya, Payload’un başlatılması (`payload.init()`) sırasında işlenir ve:

- Veritabanı bağlantısı,  
- Koleksiyon (collection) ve global şemaları,  
- Yetkilendirme ve erişim kontrolü,  
- Admin paneli özelleştirmeleri,  
- Eklentiler,  
- Yerelleştirme,  
- Dosya yükleme,  
- E‑posta ve derin bağlantılar (deep links) gibi **tüm sistem ayarlarını** belirler.

Payload v3’te bu dosya, Local API, yeni veritabanı adaptörleri (Postgres, SQLite, MongoDB) ve gelişmiş auth yapısı gibi yenilikleri de kapsayacak şekilde genişlemiştir.

---

### `payload.config.ts` İçinde Yönetilen Ana Konfigürasyon Başlıkları

Aşağıda, Payload v3 dokümantasyonuna dayalı olarak yapılandırabileceğiniz ana bölümler ve en sık kullanılan örnekler listelenmiştir.

#### 1. **Koleksiyonlar (`collections`)**
Veritabanında belge (document) olarak saklanacak içerik türlerini tanımlar.  
```ts
import { CollectionConfig } from 'payload'

const Posts: CollectionConfig = {
  slug: 'posts',
  fields: [
    { name: 'title', type: 'text', required: true },
    { name: 'content', type: 'richText' },
  ],
}

export default buildConfig({
  collections: [Posts],
  // ...
})
```

#### 2. **Globaller (`globals`)**
Tekil, site geneli veriler (örn. site başlığı, navigasyon) için kullanılır.  
```ts
const Header: GlobalConfig = {
  slug: 'header',
  fields: [
    { name: 'logo', type: 'upload', relationTo: 'media' },
    { name: 'navItems', type: 'array', fields: [ /* ... */ ] },
  ],
}
```

#### 3. **Veritabanı (`db`)**
Payload v3’te birden fazla veritabanı adaptörü desteklenir.  
```ts
import { postgresAdapter } from '@payloadcms/db-postgres'

db: postgresAdapter({
  pool: {
    connectionString: process.env.DATABASE_URI,
  },
})
```
Alternatifler: `@payloadcms/db-sqlite`, `@payloadcms/db-mongodb`.

#### 4. **Admin Paneli (`admin`)**
Admin arayüzünün bileşenlerini, kullanıcı deneyimini ve marka öğelerini özelleştirir.  
```ts
admin: {
  components: {
    beforeLogin: [CustomLoginBanner],
    afterDashboard: [CustomDashboardWidget],
  },
  meta: {
    titleSuffix: ' | Şirket CMS',
    favicon: '/assets/favicon.svg',
  },
  user: 'users', // Hangi koleksiyon admin girişi için kullanılacak
}
```

#### 5. **Eklentiler (`plugins`)**
Resmî veya topluluk eklentilerini entegre eder.  
```ts
import { formBuilderPlugin } from '@payloadcms/plugin-form-builder'

plugins: [
  formBuilderPlugin({}),
]
```

#### 6. **Yetkilendirme ve Erişim Kontrolü (`access`, `cors`, `csrf`)**
Koleksiyon, global ve admin düzeyinde erişim kuralları tanımlanır.  
```ts
import { Access } from 'payload'

const isAdmin: Access = ({ req: { user } }) => user?.role === 'admin'

export default buildConfig({
  collections: [
    {
      slug: 'posts',
      access: {
        read: () => true,
        create: isAdmin,
        update: isAdmin,
        delete: isAdmin,
      },
      fields: [ /* ... */ ],
    },
  ],
  cors: ['https://frontend.com'],
  csrf: ['https://frontend.com'],
})
```

#### 7. **Yerelleştirme (`localization`)**
Çok dilli içerik yönetimi için kullanılır.  
```ts
localization: {
  locales: ['en', 'tr', 'de'],
  defaultLocale: 'en',
  fallback: true,
}
```

#### 8. **Dosya Yükleme (`upload`)**
Dosya depolama adaptörü ve boyut limitleri gibi ayarlar.  
```ts
upload: {
  limits: {
    fileSize: 5 * 1024 * 1024, // 5MB
  },
  // v3'te adaptörler `@payloadcms/plugin-cloud-storage` gibi eklentilerle yönetilir
}
```

#### 9. **E‑posta (`email`)**
Sistem e‑postaları için SMTP veya mock adaptörü.  
```ts
import { nodemailerAdapter } from '@payloadcms/email-nodemailer'

email: nodemailerAdapter({
  defaultFromAddress: 'info@site.com',
  transportOptions: {
    host: process.env.SMTP_HOST,
    port: 587,
    auth: { user: process.env.SMTP_USER, pass: process.env.SMTP_PASS },
  },
})
```

#### 10. **Derin Bağlantılar / Derin Link (`deepLink`)**
Admin paneldeki doğrudan bağlantıların davranışını belirler.  
```ts
deepLink: {
  basePath: '/admin',
}
```

#### 11. **Hooks (`hooks`)**
Koleksiyon veya global düzeyinde tanımlanabilen yaşam döngüsü kancaları.  
```ts
hooks: {
  beforeChange: [(args) => { /* ... */ }],
  afterRead: [(args) => { /* ... */ }],
}
```

#### 12. **Uç Noktalar (`endpoints`)**
Özel REST uç noktaları eklemek için kullanılır.  
```ts
endpoints: [
  {
    path: '/custom-endpoint',
    method: 'get',
    handler: (req, res) => { /* ... */ },
  },
]
```

#### 13. **GraphQL (`graphQL`)**
GraphQL API’sini devre dışı bırakma veya özelleştirme.  
```ts
graphQL: {
  disable: true,
  // ... diğer seçenekler
}
```

#### 14. **Rota / Yol (`routes`)**
API ve admin rotaları için ön ekler.  
```ts
routes: {
  admin: '/dashboard',
  api: '/api/v2',
}
```

#### 15. **Sunucu (`serverURL`)**
Ortam değişkenlerinden gelen, medya URL’leri ve e‑posta bağlantıları için kullanılan tam sunucu adresi.  
```ts
serverURL: process.env.NEXT_PUBLIC_SERVER_URL || 'http://localhost:3000',
```

#### 16. **Telemetri (`telemetry`)**
Anonim kullanım verilerinin Payload’a gönderilmesini kontrol eder (varsayılan `true`).  
```ts
telemetry: false,
```

#### 17. **Keskin / Keskin Kenarlar (`sharp`)**
Resim işleme için `sharp` kütüphanesi ayarları.  
```ts
sharp: {
  // opsiyonel sharp konfigürasyonu
},
```

#### 18. **Tip Oluşturma (`typescript`)**
Tip üretimi ile ilgili ayarlar (v3’te `payload.config.ts` üzerinden tip üretimi yapılabilir).  
```ts
typescript: {
  outputFile: 'src/payload-types.ts',
},
```

#### 19. **Özel Bileşenler (`components`)**
Admin panelinde kullanılacak özel React bileşenlerini kaydetmek için.  
```ts
components: {
  views: {
    Dashboard: CustomDashboardView,
  },
  graphics: {
    Logo: CustomLogo,
    Icon: CustomIcon,
  },
}
```

#### 20. **Kullanıcı (`user`)**
Admin panel girişleri için kullanılacak koleksiyonun `slug` değeri.  
```ts
user: 'admins',
```

---

### Mimari İçindeki Yeri
`payload.config.ts`, Payload uygulamasının **tek ve zorunlu yapılandırma dosyasıdır**. Payload başlatılırken (`payload.init()`) bu dosyadan dışa aktarılan `buildConfig` nesnesi okunur ve:

- Veritabanı bağlantısı kurulur,  
- Local API ve REST/GraphQL uç noktaları oluşturulur,  
- Admin paneli derlenir,  
- Hook’lar, erişim kontrolleri ve eklentiler kaydedilir.

Yani bu dosya, **Payload CMS’nin omurgasıdır**; tüm modüller (koleksiyonlar, globaller, admin, API) bu merkezî konfigürasyondan beslenir. v3 ile birlikte Local API’nin doğrudan bu konfigürasyon üzerinden kullanılabilmesi, dosyanın önemini daha da artırmıştır.

---

**Özetle:** `payload.config.ts` dosyası, Payload v3 projenizin **tek gerçek kaynağı (single source of truth)** olarak çalışır. Yukarıdaki başlıklar altında yapılan her tanım, CMS’inizin veri modelini, güvenlik duvarlarını, admin deneyimini ve entegrasyonlarını şekillendirir.
